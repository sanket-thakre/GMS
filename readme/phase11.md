## 📋 Phase 11: Grievance Registration API (Backend)

- **Target Developer Assignment:** Stream 2: Complainant Portal (backend foundation for the submission form in Phase 12).
- **Primary Objective:** Create the authenticated endpoint that lodges a new grievance ticket, auto-generates a unique ticket number, computes the SLA `due_date`, supports optional file attachments, and writes the first audit-log row.

### ⚙️ Backend Requirements (FastAPI & SQLAlchemy)

Reuse `Ticket` (`ticket_number` unique, `complainant_id`, `subcategory_id`, `assigned_hierarchy_id`, `status` Enum default `Open`, `priority` Enum default `Medium`, `description`, `created_at`, `due_date`), `GrievanceSubcategory.sla_hours`, `AuditLog` (`action_type` Enum incl. `Created`), and `get_current_user`/`get_db` from `app/api/deps.py`.

**New model `TicketAttachment` (`app/models/attachments.py`):**
- `id` PK, `ticket_id` FK→`tickets.id`, `file_name` String, `file_path` String, `content_type` String, `uploaded_at` DateTime `server_default=func.now()`.
- Relationship `ticket = relationship("Ticket", back_populates="attachments")`; add the reciprocal `attachments` relationship on `Ticket`.
- Import it in `app/db/base.py` and `app/models/__init__.py`, then run `alembic revision --autogenerate -m "add ticket attachments" && alembic upgrade head` (DB is on `localhost:5433`).

**Helpers (`app/core/ticketing.py`):**
- `generate_ticket_number(db) -> str`: format `GMS-YYYY-NNNNNN` (year + zero-padded incrementing count). Derive the sequence from `db.query(Ticket).count() + 1`; loop-retry on the rare unique-constraint clash.
- `compute_due_date(created_at, sla_hours) -> datetime`: `created_at + timedelta(hours=sla_hours)`.

**Pydantic schemas (`app/schemas/ticket.py`):**
- `TicketCreate`: `subcategory_id: int`, `description: str`, optional `priority: TicketPriority = Medium` (import enum from `app.models.tickets`). **Note:** the complainant is taken from the token, never the payload.
- `AttachmentOut`: `id`, `file_name`, `content_type`, plus a computed `url` (e.g. `/static/uploads/<file>`).
- `TicketOut`: `id`, `ticket_number`, `status`, `priority`, `description`, `subcategory_id`, `assigned_hierarchy_id`, `complainant_id`, `created_at`, `due_date`, `attachments: list[AttachmentOut] = []` (`from_attributes`).

**Endpoints (`app/api/v1/endpoints/tickets.py`, prefix `/tickets`):**
- `POST /` (auth: `Depends(get_current_user)`) — accept **`multipart/form-data`** so files can ride along. Use FastAPI `Form(...)` for `subcategory_id`, `description`, `priority` and `files: list[UploadFile] = File(default=[])`. Logic:
  1. Validate the subcategory exists → 404 if not.
  2. `ticket_number = generate_ticket_number(db)`.
  3. **Assignment placeholder:** set `assigned_hierarchy_id` to a default/triage office for now (e.g. first APMC office, or `None`); Phase 14 replaces this with the real Assignment Engine. Document this clearly with a `# TODO(phase14)` comment.
  4. `due_date = compute_due_date(created_at, subcategory.sla_hours)` — since `created_at` is server-set, compute `datetime.now(timezone.utc) + timedelta(...)` for the stored due date.
  5. Persist the ticket, then save each uploaded file under `backend/uploads/` with a UUID-prefixed filename, create `TicketAttachment` rows.
  6. Write an `AuditLog` row: `action_type=Created`, `action_by_user_id=current_user.id`, `new_state="Open"`.
  7. `db.commit()`, return `TicketOut` (201).
- `GET /{ticket_id}` (auth) — return a single ticket the caller is allowed to see (complainant owner OR an officer/admin). Ownership: `ticket.complainant_id == current_user.id` or `current_user.role.name in ("Admin","DoM_Admin","DDR_Officer","APMC_Officer")`. 403 otherwise, 404 if missing. (Full listing/filtering is Phase 13.)

**Static files (`app/main.py`):**
```python
from fastapi.staticfiles import StaticFiles
import os
os.makedirs("uploads", exist_ok=True)
app.mount("/static/uploads", StaticFiles(directory="uploads"), name="uploads")
```

Add `python-multipart` to `requirements.txt` (required for `UploadFile`/`Form`).

### 🛡️ Verification & Testing Checklist

1. **Swagger** — As a Complainant token, `POST /tickets` with a subcategory whose `sla_hours=12`; confirm the response `due_date` is exactly 12 hours after `created_at`, and `ticket_number` matches `GMS-YYYY-NNNNNN`.
2. **Uniqueness** — Create two tickets back-to-back; confirm the numeric suffix increments and is unique.
3. **File upload** — Attach a JPG; confirm a row in `ticket_attachments`, a file under `backend/uploads/`, and that the returned `url` loads at `http://localhost:4010/static/uploads/...`.
4. **Audit trail** — Confirm exactly one `audit_logs` row with `action_type="Created"` and `action_by_user_id` = the complainant. A second user fetching `GET /tickets/{id}` they don't own (and aren't an officer for) gets **403**.
